# The "Big Three" of C++ Memory
## Stack: 
Automatic memory. Fast, but limited size. Used for local variables.

## Heap: 
Dynamic memory. Manual management using new and delete. Risk of memory leaks if forgotten.

## Static/Global: 
Memory allocated for the lifetime of the program.

# The C++ Compilation Pipeline
The process is generally broken down into four distinct stages:

- Preprocessing: Processes directives starting with # (like #include and #define). It literally copies the content of header files into your source file.
- Compilation: The compiler translates the preprocessed code into Assembly code specific to the target processor architecture.
- Assembly: The assembler turns the assembly code into Object Files (.obj or .o). These are machine code but aren't executable yet because they contain "holes" where external functions are called.
- Linking: The linker plugs those "holes." It connects your object files with libraries (like the STL) to produce a single executable.

# VTable
Since C++ is a statically typed language, it usually decides which function to call at compile-time. However, virtual functions enable Runtime Polymorphism.

To achieve this, the compiler implements a mechanism called the vTable (Virtual Method Table).

- vTable Creation: For every class that has at least one virtual function, the compiler creates a static table (the vTable). This table contains a list of function pointers pointing to the most derived versions of the virtual functions.
- vPtr (Virtual Pointer): Every object of that class is given a hidden pointer (usually at the very beginning of its memory layout) called the vptr.
- Initialization: When an object is constructed, the constructor sets the vptr to point to the vTable of that specific class.
- The Call: When you call a virtual function via a pointer (e.g., Shape* s = new Circle(); s->draw();), the code doesn't jump to a fixed address. Instead, it:
    - Follows the vptr to the vTable.
    - Looks up the correct function address in the table.
    - Jumps to that address.

Note: This adds a tiny bit of overhead (one extra memory lookup), which is why C++ functions aren't virtual by default—you only pay for what you use.
