带路党”——**虚函数表指针（vptr）**。


---

### 1. 这种变化的根源：vptr

当你给类加上第一个 `virtual` 函数时，编译器为了实现多态，必须在运行时知道该调用哪个版本的函数。为此，它会做两件事：

1. **生成虚函数表（vtable）**：每个包含虚函数的**类**，在内存中都有一张只读的表，记录了该类所有虚函数的地址。
2. **插入虚指针（vptr）**：每个该类的**对象**，都会被编译器自动添加一个隐藏的指针成员。这个指针指向该类对应的虚函数表。

### 2. 对象大小的变化

假设我们在 64 位环境下（指针占用 8 字节）：

```cpp
class Empty {
    int a;
}; // sizeof(Empty) = 4

class Virtual {
    int a;
    virtual void func() {}
}; // sizeof(Virtual) = 16 (4字节数据 + 8字节指针 + 4字节内存对齐)

```

**为什么是 16 而不是 12？**
这就涉及到 **内存对齐（Memory Alignment）**。CPU 处理数据时喜欢整齐排列，64 位指针要求按 8 字节对齐，所以 `int` 后面会被填充 4 个字节，以保证整个对象的大小是 8 的倍数。

---

### 3. 多重继承下的“膨胀”

如果一个类继承了多个含有虚函数的父类，情况会更复杂。这个子类对象内部会包含**多个**虚指针，每个指针对应一个父类的虚函数表。

* **单继承**：对象里通常只有 1 个 vptr。
* **多重继承**：如果你继承了 3 个有虚函数的父类，你的对象里就会有 3 个 vptr，体积直接飙升 24 字节（64 位系统）。

---

### 4. 为什么要付出这个空间代价？

既然对象变大了，为什么我们还要用虚函数？

* **解耦**：如果没有这多出来的 8 字节指针，你就无法写出 `Base* b = new Derived(); b->show();` 这种动态逻辑。
* **灵活性**：这 8 字节换来的是整个面向对象编程的核心——**多态**。

### 5. 一个特殊的例子：空类

顺便提一个冷知识，C++ 中即使是完全空的类，`sizeof` 也是 **1**。

```cpp
class Nothing {}; // sizeof(Nothing) = 1

```

这是因为 C++ 要求每个对象在内存中必须有**唯一的地址**。如果大小是 0，那两个 `Nothing` 对象的地址就会重叠。

---

### 总结

对象变大，是因为编译器在对象内存的最前端（或末尾，取决于实现）插入了 **`vptr`**。它就像是对象的“身份证”，指示了在运行时该去哪张表查找正确的函数。
